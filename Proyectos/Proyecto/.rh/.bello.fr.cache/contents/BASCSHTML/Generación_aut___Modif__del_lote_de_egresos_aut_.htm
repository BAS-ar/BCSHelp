Modificación del lote de egresos automáticos Estas modificaciones pueden efectuarse todas las veces que se desee mientras los comprobantes se encuentren en el lote provisorio. Una vez cerrado el lote, ya no podrán efectuarse modificaciones a los mismos. Al ingresar en pantalla el número correspondiente al egreso de mercadería a modificar, se exhibirán todos los datos del mismo permitiendo su consulta y modificación. Pueden agregarse nuevas líneas al comprobante, eliminarse líneas de ítems que no se deseen remitir o modificar las líneas existentes o algunos de los demás datos del comprobante. Los datos a ingresar para efectuar modificaciones a los egresos de mercadería automáticos son:   Número Número provisorio del egreso de mercadería que se desea modificar. Debe ingresarse el número de un egreso existente en el lote. Puede ingresarse el número en forma manual o seleccionarse de la lista de comprobantes que se exhibe pulsando desde este campo la tecla F3. Fecha Fecha del egreso de mercadería que se desea modificar. Destino Tipo de destinatario del egreso de mercadería que se desea modificar. Las opciones son: Cliente Depósito Agente Dependiendo del tipo de destinatario, se exhibirá un código de cliente, depósito o agente. Al precargar los datos del cliente se exhibe un tooltip con el nombre de fantasía del cliente, de no contar con dicho dato, se observará la razón social. En el caso de agentes exhibirá el nombre del mismo. Si el egreso de mercadería que se está modificando se generó a partir de uno o más pedidos internos, el destino será depósito o agente, dependiendo del destinatario de el/los pedido/s internos remitidos. Si no, el destino será cliente. Depósito Depósito del egreso de mercadería que se desea modificar. En caso que el depósito correspondiente al egreso permita stock negativo provisorio y tenga la opción No avisar de stock insuficiente de la Tabla Depósitos/Plantas destildada, se da un aviso al usuario informándolo sobre la insuficiencia de existencias del ítem en cuestión. En caso contrario (opción de "No avisar de stock insuficiente" tildada) no se emite ningún aviso. Esto es válido para ítems con y sin colores y medidas, y que manejen una y dos unidades de medida. Concepto Código del concepto asociado al egreso de mercadería que se desea modificar. Lote Tipo Código del lote al cual corresponde el comprobante seleccionado para modificar. Fecha Fecha del lote al cual corresponde el comprobante seleccionado para modificar. Liberar cantidades no entregadas Si se tilda esta opción es posible modificar el lote de egresos automáticos de modo que permita liberar las cantidades no entregadas de pedidos de venta y de pedidos internos referenciados en dicho lote.   Ficha Ítems Tabla de Ítems En esta tabla se exhiben, uno a uno, los ítems que conforman el egreso de mercadería seleccionado para modificar. Es posible agregar nuevos ítems a esta tabla informando para cada ítem la cantidad que se remite y, en función al tipo de ítem del que se trate, se requerirá información adicional. Por ejemplo, para algunos bienes se deben detallar las cantidades por color y por medida . Para aquellos bienes donde cada unidad se identifica con un número de serie, se deben informar los números de serie asociados a cada una de las unidades que egresan. Para los bienes que llevan asociado un código de partida se debe informar el código de la partida correspondiente. La misma debe haberse definido con anterioridad en la tabla destinada a tal fin y debe tener registrado el ingreso del bien que se está facturando. Para los ítems que son conjuntos, es decir, los que están compuestos por un conjunto de ítems que conforman el mismo con distintas cantidades, al entregarlos, se actualizan las existencias de los ítems componentes. Es importante tener en cuenta que la forma en que el sistema solicita los datos a incluir en el comprobante varía según cuál sea el tipo de bienes de los que se trate. En algunos casos, como por ejemplo cuando se trabaja con doble unidad de medida o con medidas y colores, se abre una ventana en el campo Cantidad para requerir los datos necesarios. En otros casos, como por ejemplo, cuando deben informarse códigos de despachos , de partidas o de centros de apropiación , se incorporan columnas adicionales para permitir incluir dicha información. Ítem Código del ítem asociado a la línea del egreso de mercadería. Si se ingresa un ítem nuevo, el código que se ingrese debe corresponder a un bien , servicio o ítem vario ingresado en el sistema. Puede ingresarse el código del ítem o su código alternativo. Descripción Descripción del ítem indicado en el dato anterior. Comprobante Se exhibe el número de comprobante al que se hace referencia en la línea del egreso. Cantidad Cantidad de unidades que egresan del ítem indicado. Aquí se presentan varias posibilidades. Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las ventas se registran en la primera unidad. Si las ventas se registran en segunda unidad, al ingresar la cantidad en la segunda unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable. Si el ítem trabaja con colores y medidas se abrirá una ventana que solicitará el ingreso de las cantidades para cada color y medida entregada, y al finalizar el ingreso de datos en esa ventana, se completará este dato con la cantidad total de unidades entregadas para todos los colores y medidas del ítem. Si el ítem es un contenedor, se abrirá una ventana que solicita el código del ítem que contiene (se ofrecerá el ítem indicado como default en la tabla de contenedores del bien) y la cantidad del mismo, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por la cantidad de unidades de este indicada en la definición del ítem contenedor. Si el ítem trabaja con dimensiones, se abrirá una ventana que solicitará el ingreso de cantidades para cada dimensión posible definida para el ítem, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por todas las dimensiones del ítem. Cantidad en 2da unidad Cantidad de unidades que se entregan del ítem indicado, en la segunda unidad de medida. Si el ítem no trabaja con doble unidad de medida, no se solicitará información en este campo. Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las ventas se registran en la segunda unidad. Si las ventas se registran en primera unidad, al ingresar la cantidad en la primera unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable. Despacho Código de hasta quince caracteres que indica el número de despacho del cual egresa el ítem indicado. Este dato podrá ingresarse para los ítems que trabajan con despachos. Debe corresponder a un despacho definido en el sistema y debe tener disponible la cantidad de unidades del ítem que se está entregando. El sistema mostrará una lista ordenada por fecha con todos los despachos en los que ingresó el ítem y podrá seleccionarse el despacho que corresponda. Partida Código de hasta quince caracteres que indica el número de partida de la cual egresa el ítem indicado. Este dato podrá ingresarse para los ítems que trabajan con partidas. Debe corresponder a una partida definida en el sistema para el ítem indicado. Se mostrará una lista con todas las partidas en las que ingresó el ítem y podrá seleccionarse la que corresponda. Mediante el botón que se observa al lado de la partida asignada, se accede a una ventana de modificación de partidas. Se muestra el código del ítem, color, medida y cantidades asignadas a cada partida y despacho (si corresponde). En dicha ventana será posible eliminar líneas de partidas o agregar nuevas partidas y despachos, de ser necesario, e indicar la cantidad de las mismas, que nunca podrá superar el stock disponible de cada partida. Al oprimir el botón OK el sistema validará que la suma de las cantidades de las partidas ingresadas en ambas unidades de medida coincida con las cantidades originales de la línea del comprobante. Si todo coincide, se procederá a modificar la línea original del ítem, agregando las nuevas líneas de las partidas indicadas en la modificación. Si el lote fue generado sin asignar partidas automáticamente, las partidas tampoco serán asignadas automáticamente en el momento de la modificación. Números de serie Si el ítem indicado trabaja con números de serie, se abrirá una ventana que permitirá el ingreso de todos los números de serie que egresan con el comprobante. Se solicitará el ingreso de tantos números como cantidad de unidades egresen del ítem. El sistema brinda la posibilidad de ingresar estos números en forma automática definiendo un prefijo común y un rango de números para los mismos. Esto es muy útil en el caso de egresos de grandes cantidades de mercadería seriada. Además, se exhibirá una lista con todos los números de serie registrados en los ingresos de mercadería del ítem, permitiendo también seleccionar de estos números cuáles son los que egresan. Centro de apropiación A Código del centro de apropiación A asociado a la línea del comprobante. Para cada ítem ingresado en el egreso de materiales, que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la de ventas o la asociada al concepto indicado para el comprobante, tiene subcuentas de tipo centro de apropiación A o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado. Centro de apropiación B Código del centro de apropiación B asociado a la línea del comprobante. Para cada ítem ingresado en el egreso de materiales, que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la de ventas o la asociada al concepto indicado para el comprobante, tiene subcuentas de tipo centro de apropiación B o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado. Ficha Entrega Esta ficha está destinada exclusivamente a exhibir información relacionada a los totales del comprobante. Los totales son calculados automáticamente. Esta ficha puede ser consultada todas las veces que sea necesario durante la modificación del comprobante y la información de la misma se actualizará en forma automática a medida que se ingresan líneas en el comprobante. Entrega en: Permite modificar el domicilio de entrega. Obs. Entrega Permite modificar el texto correspondiente a las observaciones. Cantidad Cantidad total de unidades de mercadería entregadas. Las cantidades se consideran en la primera unidad de medida, o sea según el valor de la columna cantidad de la tabla de ítems. 2da unidad Cantidad expresada en la segunda unidad de medida. Peso Peso total de la mercadería entregada expresado en la unidad indicada en los Parámetros de Ítems como unidad de peso. El sistema calcula el peso total sumando el peso unitario de cada bien (indicado en la definición del ítem) remitido por la cantidad entregada de cada uno. Las cantidades de cada bien se consideran siempre según la primera unidad de medida, o sea según el valor de la columna cantidad de la tabla de ítems. Para los ítems con contenedores el peso se calcula como la suma del peso por las unidades del ítem más el peso por las unidades del contenedor. Volumen Volumen total de la mercadería entregada expresado en la unidad indicada en los Parámetros de Ítems como unidad de volumen. El sistema calcula el volumen total sumando el volumen unitario de cada bien (indicado en la definición del ítem) remitido por la cantidad entregada de cada uno. Las cantidades de cada bien se consideran siempre según la primera unidad de medida, o sea según el valor de la columna cantidad de la tabla de ítems. Para los ítems con contenedores el volumen se calcula como el volumen del contenedor por las unidades del mismo. Bultos Número que indica la cantidad de bultos asociada al egreso de mercadería. Valor declarado Si se parametriza una lista para valor declarado (Configuración/ Parámetros/ Items), el sistema calcula automáticamente en dicho campo el valor del importe en moneda local de los ítems bienes con entrega, multiplicando el precio del ítem en la lista indicada por la cantidad en la unidad de venta. Cabe destacar que para los ítems con medidas y colores que se indican por explosión en grilla de medidas y colores, se utilizará el precio máximo para dicho ítem. El valor calculado para el valor declarado puede ser modificado por el usuario. No se muestra dicho valor para las notas de crédito o comprobantes sin entrega o remitos diferidos o comprobantes de venta con remito diferido asociado. Ficha Observaciones En esta ficha se muestran algunos datos generales del egreso de mercadería. Observaciones Texto de ingreso libre a incluir en el comprobante.

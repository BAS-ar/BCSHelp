Ingreso Esta tarea permite ingresar cotizaciones. Comprobantes_referenciados_cotizaciones Si se desea ingresar una cotización, sin precarga de datos, se debe registrar, además de la fecha y el número de comprobante:   Número externo Por ser estos comprobantes externos al sistema, se prevé la posibilidad de indicar el número original con el que fueron generados por el proveedor titular de la cotización. Este número está compuesto por el prefijo y el número propiamente dicho, y el mismo será visualizado en todos los informes en los que se incluyan los datos de la cotización. Proveedor Código del proveedor que emite la cotización. Debe ingresarse el código de un proveedor definido en el sistema. Al precargar los datos del proveedor se exhibe un tooltip con el nombre de fantasía del mismo, de no contar con dicho dato, se observará la razón social. Fecha de expiración Fecha de expiración del presupuesto que se está ingresando. Pasada esta fecha, se considera que pierde validez la oferta del proveedor.   Condición de venta Código de la condición de venta del proveedor para la que se registra la cotización. El sistema permite registrar un comprobante de cotización por cada oferta que realice el proveedor. Así, si ofrece tres precios diferentes en función a tres condiciones de pago diferentes, registraremos tres comprobantes de cotización, uno para cada condición. Moneda Moneda en la cual se ingresan los valores en la cotización. Debe corresponder al código de una tabla de tipo Cotización ingresada en el sistema. Si no se ingresa una moneda en particular, se asumirá que los importes se ingresan en moneda local.   Documentado Indicación de si existe o no un documento escrito que avale la información que se está registrando en el sistema.   Ficha Ítems Tabla de Ítems Mediante esta tabla se ingresan, uno a uno, los ítems que conforman la cotización del proveedor. Para cada bien incluido en el presupuesto, se podrá informar el precio unitario presupuestado, la cantidad de unidades a comprar y el importe total resultante. En muchos casos, el precio presupuestado depende de la cantidad de unidades a comprar. Efectivamente, muchas veces el precio varía en función al volumen de la compra; para esos casos, entonces, es que se ha incluido la posibilidad de informar la cantidad de unidades a comprar. Item Código del ítem a incluir en el presupuesto del proveedor. Debe corresponder a un bien , servicio o ítem vario ingresado en el sistema. Puede ingresarse el código del ítem o su código alternativo. Descripción Descripción del ítem indicado en el dato anterior. Si el ítem tiene asociada una descripción larga, se exhibirá esta. Si no, se exhibirá la otra descripción. Si el ítem no tiene descripción asociada, se solicitará el ingreso de un detalle del mismo en este momento. Cantidad Cantidad de unidades a comprar del ítem indicado. Aquí se presentan varias posibilidades. Si el ítem trabaja con colores y medidas se abrirá una ventana que solicitará el ingreso de las cantidades para cada color y medida presupuestada, y al finalizar el ingreso de datos en esa ventana, se completará este dato con la cantidad total de unidades presupuestadas para todos los colores y medidas del ítem. Si el ítem es un contenedor, se abrirá una ventana que solicita el código del ítem que contiene (se ofrecerá el ítem indicado como default en la tabla de contenedores del bien) y la cantidad del mismo, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por la cantidad de unidades de este indicada en la definición del ítem contenedor. Si el ítem trabaja con dimensiones, se abrirá una ventana que solicitará el ingreso de cantidades para cada dimensión posible definida para el ítem, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por todas las dimensiones del ítem. Precio Precio unitario que ofrece el proveedor para el ítem indicado. Si el ítem trabaja con colores y medidas, en la ventana donde se ingresaron las cantidades, podrán ingresarse también los precios para cada color y medida. Esto es útil si se manejan distintos precios por medida o por color. Si se utiliza un solo precio, podrá indicarse el mismo en este dato. Si estando en esta columna se oprimir SHIFT + F3, se habilita una ventana que permite, a partir del precio ingresado, indicar en una tabla porcentajes de descuento y/o recargo, a partir de los cuales se consigue un precio final. Al oprimir el botón de OK o F10 dicho precio se copiará a la grilla de ítems. Si se vuelve a oprimir SHIFT + F3 el sistema habilita la misma ventana pero no hay un registro de los descuentos aplicados anteriormente, siempre se parte del precio indicado hasta el momento en la grilla, pero se puede modificar el precio original desde la ventana que se habilita. Importe Importe del ítem presupuestado dependiendo de la cantidad de unidades a comprar del mismo. Ficha Observaciones Observaciones Permite ingresar un texto aclaratorio. Este aparece en todos los informes en los que se incluye al comprobante.   Ficha Adjuntos Permite adjuntar archivos externos. Una vez grabado el comprobante la administración de los archivos adjuntados se realiza desde el Proceso de Adjuntos. //
